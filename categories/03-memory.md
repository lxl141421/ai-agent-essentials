# 记忆系统

> 让 Agent 记住你是谁、之前做过什么、你的偏好和习惯。

---

## 为什么需要记忆？

没有记忆的 Agent 每次对话都是从零开始：
- 不知道你的名字和角色
- 不记得上次帮了你什么
- 不了解你的技术栈和偏好
- 每次都要重复说明上下文

## 记忆的层次

```
短期记忆（会话内）
  └── 最近几轮对话，上下文窗口自动管理

长期记忆（跨会话）
  └── 用户画像、偏好、环境信息
  └── 需要专门的存储和检索机制

知识库（结构化）
  └── 文档、代码库、专业知识
  └── RAG 系统管理（见知识库章节）
```

## 推荐方案

### 🥇 mem0

| 项目 | 详情 |
|------|------|
| GitHub | [mem0ai/mem0](https://github.com/mem0ai/mem0) |
| Star | ~57k |
| 协议 | Apache 2.0 |
| 语言 | Python |

**为什么选它：**
- 通用记忆层，LLM 无关
- 自动从对话中提取重要信息
- 支持向量搜索 + 图数据库
- 可以对接任何 LLM 框架

**安全评估：**
- 数据：⚠️ 可配置（本地部署或云端）
- 审查：✅ 大量用户，代码公开

---

|------|------|
| GitHub | [volcengine/OpenViking](https://github.com/volcengine/OpenViking) |
| Star | ~25k |

**为什么选它：**

**安全评估：**
- 数据：🔒 数据本地存储
- 审查：✅ 大厂维护，代码公开

---

### 🥉 内置轻量方案

大多数 Agent 框架自带简单的记忆功能：

| 框架 | 内置记忆方案 |
|------|------------|
| Hermes Agent | SQLite + 内存注入 |
| LangChain | ConversationBufferMemory |
| LlamaIndex | ChatMemoryBuffer |

**适合场景：** 需求简单，不想引入额外依赖。

## 安全提醒

⚠️ **记忆数据包含你的对话历史和个人信息**

- 确认存储位置（本地 vs 云端）
- 查看数据保留策略
- 敏感信息不要让 Agent 记住
- 定期清理不需要的记忆

---

## 决策树

```
你需要什么级别的记忆？
│
├── 只需要当前对话不丢失
│   └── → 框架内置记忆（免费、零配置）
│
├── 需要跨会话记住用户偏好
│   └── → mem0（功能全面、社区活跃）
│
├── 需要统一管理记忆 + 技能 + 文件
│   └── → OpenViking（字节出品、设计先进）
│
└── 隐私敏感，数据不能出本机
    └── → 本地方案（SQLite / 本地向量数据库）
```

# 终端 & 文件工具

> Agent 的手和脚——能操作真实世界的文件系统和命令行。

---

## 为什么是必备？

没有终端和文件工具的 Agent 只是个聊天机器人。有了它们，Agent 才能：
- 执行 shell 命令
- 读写文件
- 搜索代码库
- 运行脚本、构建项目

## 核心能力清单

| 能力 | 说明 | 风险等级 |
|------|------|---------|
| 命令执行 | 运行任意 shell 命令 | 🔴 高 |
| 文件读取 | 读取文件内容 | 🟢 低 |
| 文件写入 | 创建/修改文件 | 🟡 中 |
| 文件搜索 | 在代码库中搜索 | 🟢 低 |
| 补丁应用 | 精确修改文件局部 | 🟡 中 |
| 进程管理 | 后台运行/终止进程 | 🟡 中 |

## 安全措施

### 必须有的安全机制

```
✅ 命令审批 —— 危险命令需要用户确认
   rm -rf /  →  ❌ 需要确认
   ls        →  ✅ 自动执行

✅ 路径限制 —— 不能访问系统关键目录
   /etc/passwd  →  ❌ 拒绝
   ~/project/   →  ✅ 允许

✅ 超时控制 —— 命令不能无限运行
   默认 180 秒超时

✅ 输出截断 —— 防止巨量输出撑爆上下文
   stdout 限制 50KB
```

## 推荐方案

### 🥇 框架内置（首选）

几乎所有 Agent 框架都内置终端和文件工具：

| 框架 | 终端 | 文件读写 | 搜索 | 补丁 |
|------|------|---------|------|------|
| Hermes Agent | ✅ | ✅ | ✅（ripgrep） | ✅（模糊匹配） |
| Claude Code | ✅ | ✅ | ✅ | ✅ |
| Codex | ✅ | ✅ | ✅ | ✅ |

**为什么选内置：**
- 不需要额外配置
- 已经和框架的安全机制集成
- 权限控制、审批流程都做好了

---

### 🥈 沙箱方案（高安全需求）

如果你需要在隔离环境中执行代码：

| 方案 | 说明 |
|------|------|
| Docker 容器 | 最成熟的隔离方案 |
| E2B | 云端沙箱，专为 AI Agent 设计 |
| Modal | Serverless 沙箱 |

```bash
# Docker 沙箱示例
docker run --rm -v ./workspace:/workspace python:3.12 \
  python /workspace/script.py
```

---

### 🥉 MCP 工具服务器

通过 MCP 协议暴露文件/终端操作：

| MCP 服务器 | 功能 |
|-----------|------|
| filesystem | 文件读写、目录操作 |
| shell | 命令执行 |
| git | Git 操作 |

**适合场景：** 需要把工具能力暴露给多个 Agent。

---

## 最佳实践

```
1. 最小权限原则 —— Agent 只给它需要的权限
2. 危险命令审批 —— rm, git reset --hard, DROP TABLE 等
3. 备份机制 —— 修改前自动备份（Hermes 的 checkpoint）
4. 日志记录 —— 所有操作留痕
5. 超时控制 —— 防止命令挂起
```

---

*工具系统通常随框架内置，不需要额外安装。安全配置是重点！*

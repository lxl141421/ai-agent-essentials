# 工具安全审计

> Agent 的工具就是它的武器。武器不安全，Agent 就不安全。

---

## 审计维度

### 1. 终端工具

**风险:** 执行任意 shell 命令，最高风险等级。

**必须有的安全机制:**
- 危险命令审批（rm -rf, git reset --hard, DROP TABLE）
- 路径限制（不能访问 /etc, /sys 等系统目录）
- 超时控制（防止命令挂起）
- 输出截断（防止撑爆上下文）

**审计要点:**
```
 是否有命令白名单/黑名单？
 是否支持沙箱执行？
 是否有操作日志？
 是否支持回滚？
```

### 2. 文件工具

**风险:** 读写文件，中等风险。

**必须有的安全机制:**
- 路径限制（不能写系统文件）
- 备份机制（修改前自动备份）
- 大小限制（防止写入超大文件）

**审计要点:**
```
 是否限制可访问路径？
 是否有文件大小限制？
 修改前是否备份？
 是否支持二进制文件？（应该不支持）
```

### 3. 浏览器工具

**风险:** 可以上网、操作网页，中等风险。

**必须有的安全机制:**
- URL 白名单/黑名单
- 下载文件扫描
- Cookie 和凭据隔离

**审计要点:**
```
 是否限制可访问的 URL？
 下载文件是否安全检查？
 登录凭据是否隔离？
 是否支持无头模式？（更安全）
```

### 4. 网络工具

**风险:** 可以发送 HTTP 请求，中等风险。

**必须有的安全机制:**
- URL 白名单
- 请求大小限制
- 超时控制

---

## 安全等级总结

| 工具类型 | 风险等级 | 关键防护 |
|---------|---------|---------|
| 终端执行 | Red | 命令审批 + 沙箱 |
| 文件写入 | Yellow | 路径限制 + 备份 |
| 浏览器 | Yellow | URL 限制 + 隔离 |
| 文件读取 | Green | 路径限制 |
| 网络请求 | Yellow | URL 白名单 |

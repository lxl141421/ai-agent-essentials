# MCP 服务器安全分级

> MCP 服务器直接暴露给 LLM，一个恶意服务器能操控你的 Agent。

---

## 分级标准

### Green - 官方认证

- Anthropic 官方出品
- 代码公开，有专人维护
- 安全问题有响应机制
- **放心使用**

已认证服务器:
- `@anthropic/filesystem` - 文件操作
- `@anthropic/github` - GitHub API
- `@anthropic/postgres` - PostgreSQL
- `@anthropic/puppeteer` - 浏览器自动化
- `@anthropic/brave-search` - 网页搜索
- `@anthropic/slack` - Slack 集成
- `@anthropic/google-drive` - Google Drive

### Yellow - 社区验证

- 100+ Star，活跃维护
- 代码被多人审查
- 至少 3 个月持续维护
- **可以使用，注意审查**

验证方法:
1. 检查 GitHub Star 和最近提交时间
2. 搜索是否有安全问题报告
3. 阅读代码，特别是网络请求和文件操作部分
4. 检查依赖项是否有已知漏洞

### Orange - 社区早期

- 新项目，Star < 100
- 代码公开但审查不足
- **先在沙箱里测试**

使用前必做:
1. 完整阅读源代码
2. 在 Docker 容器中运行
3. 限制文件系统访问权限
4. 监控网络请求

### Red - 未验证

- 无人使用 / 代码不公开 / 可疑行为
- **不要使用**

红旗信号:
- 没有 README 或文档
- 代码混淆或加密
- 要求过多权限
- 联系方式缺失
- 发布历史异常

---

## 常见攻击向量

### 1. Prompt 注入

恶意 MCP 服务器在返回结果中嵌入指令，操控 Agent 行为。

**防御:** 对 MCP 返回内容做清洗，不要直接信任。

### 2. 数据泄露

MCP 服务器在处理请求时，将数据发送到外部服务器。

**防御:** 监控网络请求，使用网络隔离。

### 3. 供应链攻击

MCP 服务器的依赖被篡改，引入恶意代码。

**防御:** 锁定依赖版本，定期审计。

### 4. 权限提升

MCP 服务器请求不必要的文件系统或网络访问权限。

**防御:** 最小权限原则，沙箱运行。

---

## 审计清单

在使用任何 MCP 服务器之前:

```
 阅读完整源代码
 检查网络请求（是否有外部调用）
 检查文件操作（是否访问敏感路径）
 检查依赖项安全
 在沙箱中测试
 限制权限
 监控运行时行为
```

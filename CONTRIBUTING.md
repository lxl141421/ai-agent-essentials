# Contributing Guide

感谢你对 AI Agent Infrastructure Survival Guide 的贡献！

## 贡献原则

### 1. 安全优先 🔒
每个推荐的工具**必须**附带安全评估：
- 开源协议（MIT / Apache 2.0 / 商业 / 其他）
- 数据是否离开本机（本地 vs 云端）
- 已知安全问题（如有）
- 最后审查日期

### 2. 宁缺毋滥 🎯
- 每个领域**最多 3 个**推荐
- 新增工具必须**替换**现有工具，不能只加不减
- 理由必须充分：为什么这个比现有的好？

### 3. 真实使用 🛠️
- 推荐的工具你**至少使用过一周**
- 附带真实的踩坑记录
- 不接受从别人博客抄来的"评测"

### 4. 踩坑记录优先 📝
- 你踩过的坑 = 别人省下的时间
- 环境差异要标明（WSL / macOS / Linux / Windows）
- 解决方案要可复现

## 如何贡献

### 添加新工具

1. Fork 本仓库
2. 在对应 `categories/` 文件中添加条目
3. 填写安全评估表：

```markdown
| 项目 | 推荐等级 | ⭐ | 说明 |
|------|---------|-----|------|
| [项目名](URL) | 🥇/🥈/🥉 | Star数 | 一句话说明 |

**安全评估：**
- 协议：MIT
- 数据：🔒 本地 / ☁️ 上云
- 审查：✅ 已审查 / ❓ 未审查
- 注意事项：xxx
```

4. 提交 PR，标题格式：`[类别] 工具名 - 简述`

### 提交踩坑记录

在 `setup/` 对应环境目录下添加，格式：

```markdown
### 问题：简短描述

**环境：** WSL2 / macOS 15 / Ubuntu 24.04
**症状：** 出了什么问题
**原因：** 为什么会这样
**解决：** 怎么修
```

### 安全审计

如果你对某个工具有安全方面的发现：

1. 在 `security/` 目录下提交审计报告
2. 包含：漏洞描述、影响范围、复现步骤、建议措施
3. 标记严重程度：🔴 高 / 🟡 中 / 🟢 低

## 代码风格

- Markdown 文件使用中文为主，关键术语保留英文
- Star 数使用近似值（如 50k），不需要精确到个位
- 日期格式：YYYY-MM-DD
- 链接必须使用 HTTPS

## 不接受的贡献

- ❌ 纯功能列表（没有安全评估）
- ❌ 未经使用的"推荐"
- ❌ 广告或付费推广
- ❌ 过时项目（6 个月无更新）
- ❌ 有已知后门或安全问题的工具

## 行为准则

- 尊重所有贡献者
- 技术讨论就事论事
- 不同意见可以并存（标清楚各自的场景）
- 安全问题私下报告，不要公开 0day

*感谢你让 Agent 开发变得更安全、更省心！*

# 贡献指南

感谢你想贡献。在提交 PR 之前，有几件事需要知道：

## 核心原则

**安全评估是必须的。** 不是走形式，是真的要检查。每个推荐的工具都需要回答这几个问题：

1. 它的开源协议是什么？（MIT / Apache / 商业 / 其他）
2. 数据会不会离开本机？
3. 有没有已知的安全问题？
4. 最后一次更新是什么时候？

**宁缺毋滥。** 每个领域最多 3 个推荐。如果你想加一个新的，必须替换掉一个现有的，并且说清楚为什么新的更好。

**真实使用。** 你推荐的工具至少用过一周。不接受从别人博客抄来的"评测"。

## 怎么贡献

### 添加新工具

Fork → 在对应的 `categories/` 文件里加条目 → 填安全评估 → 提交 PR。

PR 标题格式：`[类别] 工具名 - 简述`

### 提交踩坑记录

在 `setup/` 对应平台目录下加。格式随意，说清楚三件事就行：
- 什么环境
- 出了什么问题
- 怎么解决的

### 安全问题

如果发现了安全问题，不要公开。用 GitHub 的 Private Security Advisory 报告。

## 不接受的

- 纯功能列表（没有安全评估）
- 没用过的"推荐"
- 广告或付费推广
- 6 个月没更新的项目

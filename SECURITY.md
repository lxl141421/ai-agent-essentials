# Security Audit Standard

## 安全审计标准

本项目对所有推荐工具进行安全评估。以下是评估框架。

## 评估维度

### 1. 代码安全

| 等级 | 标准 |
|------|------|
| ✅ 审计通过 | 经过专业安全审计，或代码被 100+ 开发者审查 |
| ⚠️ 需注意 | 有已知安全注意事项（非致命），使用时需注意配置 |
| ❓ 未审计 | 尚未进行系统性安全审查 |

### 2. 数据流向

| 标记 | 含义 |
|------|------|
| 🔒 数据本地 | 所有数据处理在本机完成，不发送到外部 |
| 🔐 端到端加密 | 数据传输加密，服务端不可读 |
| ☁️ 数据上云 | 数据会发送到第三方服务器 |
| ⚠️ 可配置 | 可选本地或云端，取决于用户配置 |

### 3. 供应链风险

- **依赖项审查：** 检查直接依赖是否有已知漏洞
- **维护者信誉：** 个人开发者 vs 公司维护 vs 社区维护
- **更新频率：** 最后更新时间，安全补丁响应速度
- **代码签名：** 发布是否经过签名验证

### 4. MCP 服务器专项

MCP 服务器直接暴露给 LLM，风险特别高：

```
🟢 官方认证    Anthropic 官方 / 大厂官方出品
               - 代码公开，有专人维护
               - 安全问题有响应机制

🟡 社区验证    100+ Star，活跃维护，多人审查
               - 代码公开
               - 至少 3 个月持续维护
               - Issues 有人回复

🟠 社区早期    新项目，需要自己审查代码
               - 代码公开
               - 自行审查后再使用
               - 不要在生产环境用

🔴 未验证      无人使用 / 代码不公开 / 可疑行为
               - 不推荐使用
               - 如果必须用，先在沙箱里跑
```

## 报告安全问题

如果你发现了安全问题：

1. **不要公开 0day** —— 先私下报告
2. 在 GitHub 上开 **Private Security Advisory**
3. 或发邮件到 [待定]
4. 我们会在 72 小时内回复

## 免责声明

本项目的安全评估基于公开信息和社区审查，**不构成专业安全审计**。
使用任何工具前，请自行评估风险。

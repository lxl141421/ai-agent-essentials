# 安全审计标准

这个仓库对每个推荐工具做安全评估。不是专业审计，但比"什么都不管"强。

## 我们检查什么

**代码安全** — 代码是否公开？有没有被多人审查？维护者是谁？

**数据流向** — 数据会不会离开本机？如果会，去了哪里？有没有加密？

**供应链** — 依赖项有没有已知漏洞？发布有没有签名？

**MCP 服务器** — 这个风险最高，因为直接暴露给 LLM。一个恶意服务器能操控你的 Agent。

## 分级

### 工具安全等级

- ✅ 审计通过 — 代码审查无已知风险
- ⚠️ 需注意 — 有安全注意事项，看具体说明
- ❓ 未审计 — 还没检查

### 数据流向标记

- 🔒 数据本地 — 不离开本机
- ☁️ 数据上云 — 经过第三方服务器
- ⚠️ 可配置 — 取决于你的设置

### MCP 服务器分级

```
🟢 官方认证    Anthropic / 大厂出品
🟡 社区验证    100+ Star，活跃维护
🟠 社区早期    新项目，自己先审查
🔴 未验证      没人用过，别碰
```

## 审计报告格式

如果你提交安全审计报告，包含这些：

- 审计了什么（工具名、版本）
- 发现了什么（高危 / 中危 / 低危）
- 数据流向
- 使用建议

## 免责声明

我们做的不是专业安全审计。使用任何工具前，请自行评估风险。
